Enigma

Language

About

I worked on a project where I created a simulation of the Enigma machine in Java. The Enigma machine was a mechanical device used to encrypt and decrypt messages during World War II, and my program allowed users to input messages and configure the machine with different rotors and reflectors in order to encrypt and decrypt those messages. I implemented a series of rotors and a reflector to create a progressive substitution cipher, using a one-to-one mapping of the alphabet onto itself to replace each letter in the message with another letter based on the current configuration of the machine.

Skills

While working on this project, I developed my programming skills in several areas. I implemented the Enigma machine's substitution cipher algorithm using arrays and lists to store and manipulate the rotors and reflectors. I also designed and implemented classes to represent the different components of the Enigma machine, including the rotors, reflector, and Enigma machine itself. These classes used object-oriented programming techniques, such as encapsulation and inheritance, to represent the relationships between the different components and to allow for reuse and customization of the Enigma machine's functionality.

Ataxx

Language

About

Attax is a board game played on a 7x7 grid with red and blue pieces. Players can either extend their own pieces by placing a new piece next to an existing one, or jump to an empty square that is no more than two rows and columns away. The goal is to have the most pieces on the board when no more moves are possible or after 25 jumps with no intervening extends. Players can also place blocks on the board to restrict movement. The winner is the player with the most pieces at the end of the game.

Skills

While working on the Attax project, I learned how to implement and use various data structures such as lists, dictionaries, and sets. I also practiced using control statements such as if-else statements and while loops to execute different actions based on certain conditions. Additionally, I learned how to design and implement an automated player using artificial intelligence techniques such as minimax and alpha-beta pruning. I also gained experience in creating a GUI interface for the game and handling user input.

Ants vs. Bees

Language

About

In Ants vs. Bees, I created a computer game where players take on the role of an ant queen and must defend their colony from invading bees. To do this, the player must carefully choose which types of ants to place in the colony, as each type has a different ability or action it can perform. The player must also deal with bees, which move through the colony and try to reach the end of the tunnel. The game is won by defeating all of the bees, and it is lost if a bee reaches the end of the tunnel or if the bees destroy the QueenAnt. The game takes place in a series of turns, during which the player can place new ants and the insects (ants and bees) take their actions.

Skills

To create this project, I focused on using an object-oriented programming paradigm and worked with several core classes, including GameState, Place, Hive, AntHomeBase, Insect, Ant, and Bee. I implemented various ant types, including HarvesterAnts and ThrowerAnts, and created a system for bees to move through the tunnel and attack ants. I also had to consider concepts such as health, food costs, and turn order to make the game functioning properly.
